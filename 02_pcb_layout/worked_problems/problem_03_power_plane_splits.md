# Problem 03: Power Plane Splits — Avoiding Signal Integrity Issues

## Problem Statement

You are designing a 6-layer PCB for a mixed-signal industrial controller. The board contains:

- An ARM Cortex-M4 microcontroller (3.3 V core, 3.3 V I/O, 1.8 V analogue supply)
- A 12-bit SAR ADC with separate AVDD (3.3 V analogue) and DVDD (3.3 V digital) pins
- A USB 2.0 Full-Speed PHY (3.3 V)
- CAN bus transceiver (5 V)

The Layer 4 power plane must accommodate four separate voltage domains:
- VCC_3V3_DIG (3.3 V, digital)
- VCC_3V3_ANA (3.3 V, analogue — physically separated from digital despite same voltage)
- VCC_1V8_ANA (1.8 V, analogue)
- VCC_5V (5 V, CAN transceiver)

**Part A:** Draw a text-based diagram of how the power plane splits should be arranged. Identify which signals are at highest risk from split crossings and explain why.

**Part B:** The USB 2.0 differential pair (D+/D–) must route from the MCU to a USB connector on the board edge. The connector is in the VCC_5V domain region of the board. Describe how to route the differential pair without crossing a power plane split in the reference plane.

**Part C:** A 1 MHz SPI bus (CLK, MOSI, MISO, CS) routes from the MCU to the ADC. The MCU pads are in the 3.3 V digital zone; the ADC SPI pads are in the 3.3 V analogue zone. The traces must cross the plane split between VCC_3V3_DIG and VCC_3V3_ANA. Quantify the signal integrity impact of crossing the split and specify the mitigation.

**Part D:** The 1.8 V analogue supply is generated by an LDO regulator located in the 3.3 V digital domain. The LDO output must route to the 1.8 V plane region. Describe the physical routing of the LDO output trace and how to prevent this from becoming an SI or EMI problem.

---

## Solution Approach

### Part A — Power Plane Split Layout

A well-designed power plane separates domains into contiguous regions with the split lines positioned where they cause minimum signal integrity disruption. The split lines should:

1. Be perpendicular to as few signal routes as possible
2. Not bisect any critical signal routing corridor
3. Be positioned so that high-frequency signals are referenced to a continuous plane (typically the GND plane on Layer 2, which should NOT be split)

**Layer 4 power plane layout:**

```
Layer 4 — Power Plane (top view)

+----------------------------------------------------+
|                                                    |
|   VCC_3V3_DIG                |VCC_5V              |
|   (MCU, USB PHY,             |  (CAN)             |
|    FPGA digital I/O)         |                    |
|                              |                    |
|--------- split line A -------|                    |
|                                                    |
|   VCC_3V3_ANA    | VCC_1V8_ANA                   |
|   (ADC AVDD,     |   (ADC analogue ref,            |
|   op-amp supply) |    instrumentation amp)         |
|                  |                                 |
|------ split B ---|-------- split C ----------------|
|                                                    |
+----------------------------------------------------+

Key:
  Split A: separates digital and analogue 3.3 V domains (horizontal)
  Split B: separates 3.3 V and 1.8 V analogue domains (vertical)
  Split C: separates 5 V CAN domain from analogue (horizontal)
```

**Highest-risk signals at split crossings:**

1. **USB D+/D– differential pair crossing split A:** USB 2.0 at 480 Mb/s (Full Speed: 12 Mb/s, but plan for High Speed at 480 Mb/s) requires 90 Ω differential impedance and continuous reference plane. Any split crossing degrades the reference continuity and generates common-mode noise on the USB bus.

2. **ADC SPI clock (SCK) crossing split A:** Even at 1 MHz, the SPI clock has fast edges (sub-nanosecond rise times). The return current for CLK crosses the split, increasing loop inductance, generating EMI, and potentially coupling into the analogue supply domain via the common impedance of the split gap.

3. **Analogue sensor signals crossing split B or C:** Any analogue signal crossing between VCC_3V3_ANA and VCC_1V8_ANA domains risks picking up switching noise from the other domain through common impedance at the split.

**The Layer 2 GND plane must NOT be split.** A solid, uninterrupted GND plane on Layer 2 provides a continuous reference for all signals regardless of which power domain they belong to. This is the primary mitigation for all cross-domain routing.

### Part B — USB 2.0 Differential Pair Routing

USB 2.0 Full Speed at 12 Mb/s has a Nyquist frequency of 6 MHz and edge rates of approximately 4 ns. The electrical requirements for board-level routing are:

```
Differential impedance: 90 Ω ± 15%
Maximum trace length: 200 mm (board-level, FS)
Intra-pair skew: < 0.5 UI = 42 ns at 12 Mb/s (very relaxed at FS)
```

**Strategy to avoid split crossing:**

The USB connector is in the VCC_5V region (CAN transceiver side). The MCU (source of the USB DP/DM signals) is in the VCC_3V3_DIG region. The differential pair route must cross the board without crossing a power plane split in its reference plane.

**Three options:**

**Option 1 (preferred): Route on a layer that references only the continuous GND plane.**

If the differential pair is routed on Layer 1 (top, microstrip) or Layer 3 (stripline between L2 GND and L4 power), it references L2 GND primarily. The L4 power plane split is still present but further away and less dominant as a reference for the upper layers.

```
Routing on L1 (microstrip):
  Reference plane: L2 GND (solid, uninterrupted)
  The L4 power split is 0.46 mm below L2 — at this distance, the field coupling
  to L4 is weak. The L2 GND plane is the primary image current path.
  Result: no split crossing in the dominant reference plane.
```

**Option 2: Route under the GND plane.**

Route the USB pair on Layer 5 or Layer 6, referenced to L6 (GND) and/or L4 (power). However, this places the pair adjacent to the power plane with splits — less desirable.

**Option 3: Add a bridge pour.**

If the routing must remain on a layer that references L4, add a 3.3 V copper pour bridge over the split gap, directly under the USB pair route. This restores AC continuity for image currents crossing the gap, at the cost of shorting two power domains — which is only acceptable if the two domains are at the same voltage (not applicable here for the 3.3 V / 5 V boundary).

**Recommended implementation (Option 1):**

Route USB D+/D– on Layer 1 as a 90 Ω edge-coupled microstrip pair (W ≈ 0.175 mm, S ≈ 0.175 mm on 0.1 mm prepreg — calculated per IPC-2141A). The pair references L2 GND throughout its full length. The split between VCC_3V3_DIG and VCC_5V on L4 does not disrupt the reference.

### Part C — SPI Bus Crossing VCC_3V3_DIG / VCC_3V3_ANA Split

**Quantifying the SI impact:**

The SPI clock at 1 MHz with sub-nanosecond edges generates harmonics to ~500 MHz. The return current for the SPI traces travels on the nearest reference plane. If that plane has a split, the return current detours around the gap.

```
Split gap width: assume 0.3 mm (typical plane split clearance)
L2 GND plane: solid — provides an alternative return path above the split

For a signal on Layer 3 (stripline), referenced to both L2 GND and L4 power:
  Distance to L2 GND: 0.36 mm
  Distance to L4 power: 0.36 mm
  The image current divides between both planes.

If L4 has a split but L2 is solid:
  The return current that would have used L4 transfers to L2.
  This increases the loop area slightly (by the distance from the trace to L2
  rather than L4), but since L2 is close, the increase is modest.
```

**For Layer 1 routing (microstrip):**

The SPI bus on Layer 1 references L2 GND only (dominant reference plane for microstrip). The L4 split has negligible direct effect. The key is maintaining L2 GND continuity under the SPI route.

**Mitigation — stitching capacitors across the analogue/digital split:**

Place two 100 nF MLCCs directly at the split line, one on each side of the SPI route crossing, bridging VCC_3V3_DIG to VCC_3V3_ANA:

```
Physical placement:
  Capacitor C1: 0.5 mm to the left of the SPI crossing point on the split line
  Capacitor C2: 0.5 mm to the right

These capacitors provide an AC short across the split at high frequency, allowing
the return current from the SPI CLK edges to cross the split through the caps
rather than taking a long detour around the split ends.

At 1 MHz: Xc = 1 / (2π × 1 MHz × 100 nF) = 1.59 Ω
At 100 MHz (10th harmonic): Xc = 0.016 Ω — effectively a short circuit

The stitching capacitors restore AC continuity without DC-shorting the two 3.3 V
supplies (which, despite being the same voltage, are intentionally kept separate
to prevent digital noise from coupling into the analogue domain via the common
power supply rail).
```

**Ferrite bead isolation:**

The analogue 3.3 V supply uses a ferrite bead (or LC pi filter) to isolate it from the digital 3.3 V supply at the LDO or regulator output. The split in the power plane reinforces this isolation — the stitching caps restore AC signal return path continuity without undermining the ferrite bead's purpose (which operates at low to mid frequency, typically 10–100 MHz cutoff).

**SPI signal routing guidelines for the crossing:**

```
1. Route all SPI signals (CLK, MOSI, MISO, CS) in a tight bundle (< 2 mm span)
   to cross the split at the same point. This minimises the number of discrete
   crossing locations and concentrates the stitching cap requirement.

2. Perpendicular crossing: routes cross the split at 90° — minimise the length
   of trace running parallel to the split, which would maximise inductive coupling
   to adjacent domains.

3. Place a solid copper GND pour on Layer 1 (top) and Layer 6 (bottom) in the
   split region to provide a surface-level return path in parallel with Layer 2.

4. After crossing, keep SPI signals away from analogue sensor traces for at
   least 5 mm to allow switching noise to dissipate before reaching
   sensitive ADC input nodes.
```

### Part D — LDO Output Routing from Digital to Analogue Domain

**Physical topology:**

The LDO regulator is in the VCC_3V3_DIG region. Its output (1.8 V) must reach the VCC_1V8_ANA plane region. The LDO output is a low-impedance source; the trace routing from the LDO to the 1.8 V plane must not couple switching noise from the digital domain into the analogue plane.

**Routing strategy:**

```
Step 1 — LDO placement:
  Place the LDO as close as possible to the boundary between the digital and
  analogue regions. The output trace crossing length is minimised.

Step 2 — LDO output LC filter:
  Even though an LDO inherently has low output ripple compared to a switching
  regulator, the LDO output trace in the digital zone can pick up switching
  noise capacitively from adjacent digital traces.

  Add an LC pi-filter or at minimum a ferrite bead between the LDO output
  and the entry point to the VCC_1V8_ANA plane:

    LDO out → Ferrite bead (60 Ω @ 100 MHz) → VCC_1V8_ANA entry
                                     |
                               100 nF + 10 µF (on analogue side)

Step 3 — Routing the output trace:
  Route the LDO output trace on Layer 3 or Layer 5 (inner layer) to shield it
  from surface-layer digital signals. Keep it as short as possible (< 15 mm).
  Do not route it parallel to digital clock traces.

Step 4 — Entry point into analogue plane:
  The 1.8 V trace enters the VCC_1V8_ANA copper pour via a via from the inner
  routing layer. This via is the only entry point for the 1.8 V supply to the
  plane. The plane then distributes the supply to the ADC analogue pins.

Step 5 — Decoupling at the plane entry:
  Place 100 nF + 10 µF directly at the entry via, on the analogue side of the
  plane border. This provides local charge reservoir and high-frequency filtering
  before the supply reaches the ADC.
```

**GND plane continuity reminder:**

All of the above assumes Layer 2 GND is solid and continuous under both the digital and analogue regions. The LDO output trace, the SPI crossing, and the USB pair all rely on the unbroken GND plane to provide a continuous low-inductance return current path. Never split the GND plane to match power domain boundaries — this is one of the most common serious errors in mixed-signal PCB design.

**Common mistake:** Splitting the GND plane under the analogue section to "prevent digital ground noise from entering the analogue domain." This actually worsens SI because it forces return currents from analogue signals to take long detours under the split. The correct approach is a single solid GND plane with careful placement and routing, not a split GND.

---

## Key Takeaways

- The GND plane (Layer 2) must never be split — it is the reference for all return currents. Only the power plane is split by voltage domain
- USB and other high-speed differential pairs should reference a continuous GND plane; route on Layer 1 (microstrip) to avoid the Layer 4 power plane split entirely
- SPI and other medium-speed signals that must cross a power domain split use stitching capacitors (100 nF placed directly at the crossing point) to restore AC return current continuity without DC-connecting the domains
- LDO outputs entering an analogue power domain should cross through a ferrite bead filter and have decoupling placed on the analogue side of the entry point
- Power plane splits must be placed perpendicular to as few signal routes as possible — plan the split geometry before routing begins, not after

## Interview Notes

This problem tests a core mixed-signal PCB design skill that is highly relevant to industrial instrumentation, medical devices, and data acquisition systems. The key insight interviewers look for is:

- "Never split the GND plane" — this is a fundamental rule that many candidates get wrong, proposing GND plane splits under analogue regions
- Stitching capacitors restore AC return path continuity across plane splits — knowing this and knowing where to place them distinguishes experienced designers
- Understanding that the same voltage (3.3 V digital vs 3.3 V analogue) still requires physical separation in the power plane, maintained by a ferrite bead filter at the boundary

A follow-up question is often: "If the ADC AVDD and DVDD are the same voltage, do they need to be separate plane regions?" The answer is yes, for EMI and noise coupling reasons — the digital switching currents must not share a common impedance with the analogue supply path. The ferrite bead provides high-frequency isolation; the plane split provides DC isolation and layout clarity.
